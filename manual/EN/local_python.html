8.1 What is Local Python (Embeddable Runtime)?

The Local Python tab is used to download and manage so-called "Windows Embeddable" versions of Python. These are official but minimized versions of Python directly from python.org.

Why use them instead of a classic installation?

  • Absolute Isolation: These versions are not installed into Windows. They do not write anything to the system registry and do not modify system variables (PATH).
  • Portability: If you create a virtual environment (venv) from Local Python, the entire structure is portable. You can copy it to a USB drive and it will work on any other computer.
  • Multi-version Testing: You can have Python 3.9, 3.10, and 3.12 downloaded side-by-side without them "clashing" in your system.

8.2 Installation and Sources (URL/ZIP)

In the upper part of the tab, you can add a new Python version to the application. You have three options:

  1. Select from Default List: In the dropdown menu, you will find pre-prepared safe download links for the most used versions directly from Python servers.
  2. Custom URL Address: If your version is missing from the list, paste a direct link to a .zip file (must start with http/https and end with .zip).
  3. Local ZIP File: If you already have the Python .zip downloaded on your PC, click the folder icon 📁 and select it. The application won't need to download it, just extract it.

Installation Process and Access Rights

The default path for storing downloaded runtimes is the shared folder C:\ProgramData\VenvHubPro\PyRuntimes. This folder is common to all computer users, preventing data duplication.

Seamless Operation without UAC: The system installer automatically unlocked access permissions for this folder during initial setup (the standard Windows users group was granted full control). This means that downloading, installing, and managing local Python versions happens completely in the background without requiring administrator (UAC) rights during daily use.

8.3 Critical Step: Repairing PIP (Must-do!)

As soon as a new version is installed and appears in the "Installed Runtimes" table, you will notice that the "PIP Status" column shows a red ❌ Missing.

Reason: The minimized "Embeddable" versions of Python from the python.org developers are intentionally "blind". By default, they do not include a package installer (Pip) and have blocked access to so-called site-packages (third-party libraries).

What do you need to do?

Click on the row in the table and press the ✅ Repair PIP button. At this moment, VenvHub Pro performs a series of extremely important "hacks" and modifications:

  1. Downloads Bootstrap: The application silently downloads the official get-pip.py script.
  2. Unlocks "Blindness" (._pth file): VenvHub Pro finds a hidden configuration file inside Python (e.g., python312._pth) and programmatically uncomments the import site line. Without this step, Python would ignore any packages you install into it.
  3. Installs PIP and Virtualenv: The script runs, installs Pip, and subsequently downloads the virtualenv package. (Embed versions do not contain the standard venv module for creating environments, so VenvHub replaces it with the more powerful virtualenv tool).
Result: After completing this process, the status in the table changes to ✅ OK. From this moment on, this "restricted" Python behaves like a full-featured, high-performance, and portable installation, and you can create environments from it in the main Manager.

8.4 Editing the Version List (JSONL)

The list of versions in the dropdown menu is not hard-coded in the application. It is read from a text file called jso_python_ver.jsonl in the assets folder. If a new version of Python is released (e.g., 3.14), you can add it to the list yourself.

  1. Click the ✏️ Edit List button.
  2. The application will open the file in Notepad. (Note: If the program is in Program Files, Notepad will request Administrator rights via UAC).
  3. Add a new record on a new line in exactly this format: {"name": "Name (e.g., Python 3.14.0)", "url": "https://link-to-zip-file.zip"}
  4. Save the file (Ctrl+S) and close Notepad.

VenvHub Pro automatically monitors changes in this file. As soon as you save it, the dropdown menu in the application will update instantly without a restart.

8.5 Management, Deletion, and Cache

Speed through Caching

To prevent lag and slow loading, VenvHub Pro does not read the list of installed Python versions and their statuses from the disk every time you switch tabs. It keeps them in fast operational memory (RAM).

How to correctly delete Python versions?

Because the application uses a Cache, never delete Python folders manually via Windows Explorer! The application wouldn't know about it, the version would still appear in the table, and the program would crash when trying to use it.

  • Always use the ❌ Remove button directly in the application. VenvHub Pro safely deletes all files from the disk and immediately clears the record from its internal memory.

What if the table "gets stuck"? (Refresh Button)

If you deleted a folder manually despite the warning, or if you added one by copying from another PC, click the 🔄 (Refresh) button under the table.

This forces a "Hard Refresh". The application discards its cache memory, physically scans the entire disk, and re-tests all found Python versions, synchronizing the table with reality.
