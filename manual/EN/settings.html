7.1 Basic Paths (Paths) and risks

The "Settings" tab in the main Manager window is used to configure key folders and application behavior. These are the same paths you set during the first launch, expanded with advanced features:

  • Projects Root: The main folder where VenvHub Pro searches for your source codes (project folders).
  • Venv Hub: The central folder where all virtual environments are physically stored and isolated.
  • Local Packages: The folder for your own shared modules, which the application dynamically links to venvs without using UAC (Linker module).
  • Local Packages (pip -e): The folder for your source codes installed in editable mode (require setup.py or pyproject.toml).
  • VS Code Profile: Selection of an active, fully isolated user profile for Visual Studio Code (custom extensions and settings).
Warning before changing paths!
The application logically links your projects with environments. If you already have environments created and change the "Venv Hub" path, the application will look into the new empty folder, find nothing, and your environments will "disappear" from the table (though they still physically exist on the old drive).

How to do it correctly? If you need to move data to another drive (e.g., from C: to D:):
  1. Close VenvHub Pro.
  2. Move the entire folder with Venvs to the new drive using Windows Explorer.
  3. Open VenvHub Pro and change the path to the new location in Settings.

7.2 Package Manager (Pip vs UV)

This is one of the most important settings, defining the application's engine under the hood. It determines which tool will be used whenever you click Install, Uninstall, Update, or when environments are created and cloned.

Standard (Pip)

The traditional, time-tested standard in Python. It is slower and cannot automatically resolve complex dependency conflicts during bulk updates. Its advantage is 100% compatibility with all systems (no additional installation required).

Ultra-fast (UV - Astral)

An extremely fast tool written in Rust that replaces Pip. Installations are often 10x to 100x faster thanks to global caching. Additionally, it unlocks the "Autopilot" feature, which can automatically repair broken environments after updates.

Important Notice: UV is not a standard part of Python. If you switch to UV but this tool is missing from your Windows system, VenvHub Pro will immediately detect it (via the internal UV Detector), display an error message (prompting you to install it with pip install uv), and safely switch the interface back to Pip to prevent application failure.

7.3 Appearance (Skins) and AI-assisted customization

VenvHub Pro supports full appearance customization using .qss (Qt Style Sheets) files, which are the equivalent of CSS for web pages.

Dual-Theme system

The application automatically searches and combines themes from two locations:

  1. Built-in: Themes hidden directly in the source code or application installation folder (e.g., VS Dark, Dracula, Solarized).
  2. Custom: Themes you have downloaded or created. They are located in the themes folder either next to the VenvHubPro.exe file (for Portable installation) or in %APPDATA%\VenvHubPro\themes.

Overriding: If you have a theme with the same name as a built-in theme in your user folder (e.g., you modify a color in Dracula.qss), the application will prioritize your file. You can thus modify any official theme without worrying about losing your changes during a program update.

Working with icons in themes

Since the application runs as a standalone compiled program, standard relative paths to images in QSS files may not work reliably. The system therefore uses a combination of dynamic system path replacement and absolute links. You have the following options for adding custom icons:

  • A) Using the built-in themes folder (Recommended for portability):
    The application's internal themes folder serves as the default storage. If you copy custom icons here (for example, into a icon/my_icons/ subfolder inside the themes folder), you can reference them universally in QSS. The skin manager will automatically search for the {{CORE_THEMES_DIR}} placeholder and replace it with the real absolute path upon loading.

    QSS entry example:
    qproperty-icon: url({{CORE_THEMES_DIR}}/icon/my_icons/save.svg);
  • B) Using an absolute path:
    If you want to keep icons in a fixed location outside the program installation, you can use a direct link (backslashes \ must be rewritten as forward slashes /).

    QSS entry example:
    qproperty-icon: url(C:/MyIcons/custom_icon.png);

Masterclass: How to replace text Emojis with real SVG icons

By default, the application uses Emojis or special characters (▶, 📁, 🔄, ⚙) on buttons in many places. You can completely replace them with professional SVG icons via QSS. However, it is critically important to distinguish what type of button it is. Otherwise, you might either delete the text or break translations!

Situation 1: Icon-ONLY buttons (e.g., Play ▶, Folder 📁, Settings ⚙)

This is simple. In QSS, we completely hide the original Emoji character by setting color: transparent !important; and simply add your SVG icon to the background.

QPushButton#btn_play {
    /* 1. Hides the text character ▶ so it doesn't overlap the new icon */
    color: transparent !important; 
    
    /* 2. Loads the real graphic icon */
    qproperty-icon: url({{CORE_THEMES_DIR}}/icon/play.svg);
    qproperty-iconSize: 16px 16px;
    
    /* 3. Clears the background */
    background-color: transparent;
    border: none;
}

Situation 2: Buttons with BOTH Icon and Text (e.g., "🔄 Refresh", "❌ Uninstall")

What to watch out for: If you were to use color: transparent here, the text (e.g., "Refresh") would also disappear. If you tried to "cheat" it and hardcode qproperty-text: "Refresh"; via QSS, you would break the application's multilingual support! VenvHub Pro uses LanguageManager – if the user switches the language to English, QSS would still forcibly keep the text in the original language.

Correct procedure:

  1. Open the language files in the translations/ folder (e.g., sk_SK.json and en_US.json).
  2. Remove the Emoji character directly there. For example, change "btn_refresh": "🔄 Refresh" to a clean "btn_refresh": "Refresh".
  3. In QSS, do not hide the text (no transparent). Simply add the icon. The QT Framework will take care of inserting the SVG icon and rendering the text in the correct language right next to it.
QPushButton#btn_refresh {
    /* We just add the icon. Text and color are handled by LanguageManager. */
    qproperty-icon: url({{CORE_THEMES_DIR}}/icon/refresh.svg);
    qproperty-iconSize: 16px 16px;
    
    /* Slight text offset from the icon for a better look */
    padding-left: 10px;
    text-align: left;
}

Situation 3: Changing the main window icon (Application Header)

The icon itself on the top left next to the application name (in your Custom Title Bar) is not a button, but a QLabel text field named lbl_icon. Therefore, qproperty-pixmap applies to it, not qproperty-icon.

When modifying it, we encounter a technical limitation: the Python core of the application explicitly prohibits automatic resizing of this element using the self.lbl_icon.setScaledContents(False) command. If you use a standard entry, large SVG icons will not shrink but will be ugly-cropped.

Solution A: Forced adjustment with fixed dimensions in QSS
The QT Framework has a great feature – using the qproperty- prefix, you can override any setting previously defined in Python within QSS. Even if Python set the icon not to adjust in the code, we "override" it in QSS by forcing the scaledContents property directly through the style and defining a fixed size: 
QLabel#lbl_icon {
    /* 1. Forcibly override the Python setting via QSS and enable pixmap scaling */
    qproperty-scaledContents: true;

    /* 2. Assign the new SVG icon */
    qproperty-pixmap: url({{CORE_THEMES_DIR}}/icon/app_icon.svg);

    /* 3. Define exact dimensions for the icon so it doesn't stretch across the bar */
    min-width: 20px;
    max-width: 200px; /* maximum width */
    min-height: 20px;
    max-height: 20px;
}

How it works under the hood without touching Python:
Thanks to qproperty-scaledContents: true;, QSS searches for the setScaledContents(true) function in the QLabel C++ core and executes it when the theme is loaded. This completely ignores and overwrites the original Python prohibition (setScaledContents(False)). Subsequently, qproperty-pixmap applies your new SVG icon, which smoothly shrinks to the exact defined dimension.

Solution B: Dynamic adjustment (Respecting dimensions from Python)
QSS can perfectly respect any dimensions set in the .py file (or in the .ui design). If you don't set any fixed widths in QSS (width, min-width, max-height), the QT Framework automatically uses the dimensions defined by the programmer in Python using: self.lbl_icon.setFixedSize(30, 30) (or any other number).

To let QSS know it should take your new SVG icon and adjust (crop/shrink) it exactly to the dimension already existing in Python, just tell it like this: 
QLabel#lbl_icon {
    /* 1. Tell QSS: "Take the image and shrink it exactly to the size set for this window in Python" */
    qproperty-scaledContents: true;

    /* 2. Load the new icon */
    qproperty-pixmap: url({{CORE_THEMES_DIR}}/icon/app_icon.svg);
    
    /* NO width or height settings (min-width, max-height) are written here! */
    /* Consequently, QSS automatically respects the 30x30 dimension (or any other) in the .py code */
}

Why is this solution ingenious and dynamic?
If you decide to change the icon size in the .py file in the future, e.g., to 40x40 or 25x25, you don't have to fix anything in QSS. QSS will read the current size from the Python memory at startup and automatically and smoothly fit your new SVG icon into it without distortion.

How to create a custom theme using AI (e.g., ChatGPT / Claude)?

Don't like any of the default themes? Create your own, it's very easy!

  1. Go to the application installation folder (where VenvHubPro.exe is). For the Portable version, you'll find the core/themes/ folder there. For the standard version, go to C:\Program Files\VenvHubPro\core\themes\.
  2. Find any .qss file, such as VS Dark.qss, copy it to your Desktop, and open it in Notepad.
  3. Copy the entire text content of the file and paste it into an AI tool with the request: "Here is a QSS style for a PyQt6 application. Change all colors to create a modern, bright 'Cyberpunk' theme. Emphasize neon colors for buttons."
  4. The AI will generate a new code for you. Important: Remember to add this header at the very beginning of the generated code so VenvHub Pro names the theme nicely: 
    /*
* Theme Name: My Cyberpunk Theme
* Version: 1.0
* Author: Your Name and AI
*/
  5. Save the file as, for example, Cyberpunk.qss.
  6. Open VenvHub Pro, go to Settings and click Import theme.... Select your new file. The application will automatically copy it to your user folder and include it in the menu!

7.4 Language Change (Language)

The application language can be changed at any time in the Quick Settings panel (by clicking ⚙️ on the Mini-Bar).

In the "Language / Jazyk" dropdown list, you will find these options:

  • Auto: The application will attempt to detect your Windows operating system language and load the corresponding translation. If not found, it defaults to English (en_US.json).
  • Other languages: The list is generated dynamically. The application scans the translations folder and loads all available .json language files. If you upload your own translation (e.g., cs_CZ.json) to this folder, it will immediately appear in the menu.

After changing the language, the entire application interface, including all open windows, is translated immediately (without requiring a restart).

How to edit a translation or create a new one using AI (e.g., ChatGPT / Claude)?

Yes, you can easily edit existing translations, fix texts, or translate the entire application into a completely new language with the help of artificial intelligence.

Important operating rule: The application assigns translations to individual controls based on their technical identifier (objectName). In JSON files, it looks like a "key": "value" pair. When making any edits, you must change only the value on the right side. The key on the left side must remain exactly in its original form; otherwise, the translation will stop working.

Procedure for creating a new translation using AI:

  1. Go to the application installation folder and open the translations/ folder.
  2. Find the default English translation en_US.json (or Slovak sk_SK.json).
  3. Open the file in Notepad, copy its entire text content, and paste it into an AI tool with this request:

    "Here is a JSON translation file for a PyQt6 application. Translate all text values (the right sides of the quoted values) into [insert desired language, e.g., French]. Keep the technical keys on the left side (e.g., widget and window names) in their original state without any changes."
  4. TIP for a nice name in the menu: If you want the language to be called nicely in the application dropdown menu (e.g., "Français" instead of the file name), add this key anywhere in your JSON file:
    "_language_name": "Français",
  5. Save the AI-generated code in the translations/ folder under a new name according to standard language codes (e.g., fr_FR.json for French).
  6. Open VenvHub Pro and go to Quick Settings. The new language will automatically appear in the menu.

Quick fix tip: If you just want to edit the translation of a specific button, just directly open the corresponding .json file, rewrite the text in quotes on the right side, and save the file.
