6.1 Target and Environment Selection

VenvHub Pro includes an integrated visual tool for converting Python scripts (.py) into standalone executable applications (.exe) for Windows. The tool acts as a wrapper for the popular PyInstaller library.

Top Section: Where to start?

  1. Project Selection: Choose the project you want to package.
  2. Source Venv: This is critical. PyInstaller will "extract" and package exactly the libraries and versions currently installed in this selected environment.
  3. Entry Script: The main file (e.g., main.py). The application automatically scans only the root folder of your project and offers all .py files found there.
  4. Output Folder: Where the finished .exe file and temporary build files will be saved. By default, this is the dist folder inside your project.
Installation Check: If the pyinstaller package itself is missing in the selected Venv, you don't need to install it manually. VenvHub Pro will detect this when you click "BUILD" and offer a quick background installation.

6.2 Tab 1: Basic Settings

Window and Console Mode

  • Show Console: Suitable for CLI (Command Line) tools or during development. If your application crashes, you will see the error output in the black console window.
  • Hide Console (Windowed / --noconsole): Intended for final graphical applications (GUI). When the .exe is launched, only your interface will appear, without the distracting black console window in the background.

Packaging Method (Performance vs. Portability)

  • One File (--onefile): Everything (your code, libraries, Python interpreter) is compressed into a single .exe file.
    • Advantage: Extremely easy distribution to users.
    • Disadvantage: Slower startup. Each time it runs, the program must first unpack its contents into a temporary Windows folder (Temp), which can take several seconds.
  • One Directory (--onedir): The application is generated as a folder where the .exe file is surrounded by hundreds of .dll and .pyd files.
    • Advantage: Instant startup. Files are already unpacked; the system just loads them into RAM.
    • Disadvantage: You must send the entire folder (preferably zipped) to the user, not just one file.

6.3 Tab 2: Adding Data (Assets)

PyInstaller is designed to search for and package only Python code (files imported via import). If your application needs images, sounds, fonts, or configuration files (e.g., config.json) to run, you must add them here manually. Otherwise, the packaged application will crash with a FileNotFoundError.

How to fill the table?

In the table, you define the mapping: "Take this file from my disk and store it in the .exe file under this name/folder."

"Source" Column "Target Folder" Column Meaning
C:\Project\logo.png . (dot) The image is copied directly next to the .exe file (to the root).
C:\Project\ui_files\ gui/templates The entire folder content is packaged, and a new gui/templates/ structure is created inside the .exe.
Critical warning for developers: After packaging (especially in OneFile mode), standard relative paths in your code like open("logo.png") will not work! Your code physically runs from a random temporary Temp folder created by Windows.

In your Python code, you must use the following trick to find the files: 
import sys
import os

def get_resource_path(relative_path):
    """Gets absolute path to resource, works for dev and for PyInstaller"""
    try:
        # PyInstaller creates a temp folder and stores path in _MEIPASS
        base_path = sys._MEIPASS
    except AttributeError:
        # We are in development mode (running from normal folder)
        base_path = os.path.abspath(".")

    return os.path.join(base_path, relative_path)

# Usage:
logo = get_resource_path("logo.png")

6.4 Unique VenvHub Pro Features

In the Builder interface, you will find three very powerful features that pure PyInstaller cannot do alone without our wrapper.

1. Bundling the Birth Certificate (Application Metadata)

This feature takes the complete package list and Python version from your environment and generates a [ProjectName]_birth_certificate.json before the build starts.

How to use it: Enter a relative path inside your project (e.g., data/metadata) into the text field. VenvHub Pro will create this file on your disk, add it to the Assets table in the background, and ensure it's packaged directly into the .exe. Your finished application can then "remember" and display when and in what environment it was compiled.

Safety Lock: The application strictly ensures you don't accidentally select a folder outside your project (e.g., on another drive). If you try, the system mathematically recalculates paths, blocks the action, and warns you. This guarantees that your project always remains safely packagable and portable.

2. Bundling the UV Installer (uv.exe)

Do you have an application that needs to dynamically download and install additional Python libraries (e.g., plugins) after installation at the client's site? Classic pip is too slow for this.

  • By checking this option, VenvHub Pro scans your active Venv environment.
  • It finds the uv.exe file.
  • It automatically adds the --add-binary argument and bundles uv.exe into the core of your resulting application.
  • If the application finds that uv.exe is missing in the Venv, it will offer to download and install it for you (requires internet).

3. Automatic Local Package Bundling (VenvHub Auto-Injector)

If you use the Local Packages (Linker) feature in your projects, you don't have to manually add dozens of hidden imports or search for paths to their static data (images, .json, .ui files) during compilation.

VenvHub Pro contains an intelligent pre-processor that does all the work for you before PyInstaller runs:

  • Automatic Code Discovery: VenvHub Pro reads the venvhub.json config of the active Venv in the background, identifies paths to linked local packages, and automatically generates the --paths parameter for their parent folders. PyInstaller will find and compile your code without issues.
  • Automatic Assets Injection: For each checked package, the system automatically generates the --add-data "C:\Path\Package;Package" parameter. This ensures all images, translations, or configurations inside your module are copied exactly where the program expects them.
⚠️ A small but important notice (outside VenvHub):

We've noticed that some developers use folder names with spaces (e.g., ai sync).

VenvHub Pro can handle this flawlessly – when generating the command for PyInstaller, it automatically and safely wraps these paths in quotes, so compilation proceeds correctly (as seen in the Live Preview).

However, watch out for Python itself: According to the PEP 8 standard, Python does not allow spaces in module names. Therefore, you wouldn't be able to write import ai sync in your code. We recommend always renaming local package folders on disk to a version with an underscore, for example, ai_sync. If it's just a typo and you actually have it right, feel free to ignore this warning.

6.5 Tabs 3 and 4: Imports and Advanced

3. Hidden Imports

PyInstaller scans your code for import statements to determine which libraries to package. However, sometimes libraries are called hiddenly or dynamically (e.g., via importlib or by other complex libraries internally).
If the resulting .exe crashes immediately upon launch with a "ModuleNotFoundError: No module named 'something'" error, enter the name of the missing module into the Hidden Imports table. PyInstaller will bundle it "by force".

3. Excluded Modules

Sometimes PyInstaller bundles huge libraries that your application doesn't use at all, unnecessarily bloating the .exe file size (this often happens with libraries like tkinter, pandas, or numpy if they are called as optional dependencies by another library). By adding their names here, you tell PyInstaller: "Ignore this library, even if someone requires it."

4. Advanced: Cache Cleaning and UAC

  • Clean Cache (--clean): We recommend keeping this enabled. It deletes temporary files in the build folder from previous attempts, preventing the packaging of outdated code.
  • Request Admin Privileges (--uac-admin): The resulting .exe file will have a small shield on its icon. When a user runs it, Windows will first prompt them to grant Administrator privileges (suitable for applications that modify system files or registries).
