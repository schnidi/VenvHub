4.1 Architecture and Loading (Dual-Scan)

The Pip Manager is a separate window that opens after a double-click or right-click on an environment in the main Manager. It utilizes an advanced Dual-Scan algorithm. To provide you with a perfect overview, the application performs two parallel processes in the background:

  1. Local Scan: Determines what is installed on your disk (pip list --format=json).
  2. Cloud Scan: Connects to PyPI servers and identifies the latest versions (pip list --outdated --format=json).

The results are combined into a single table. If the version number is GREEN, you have the latest version. If it is RED, a newer update exists.

Intelligent Fallback: For absolute stability (and to ensure 100% compatibility with the upcoming Python 3.14), the scanning of the package list always occurs via the standard Pip module. Even if you have the UV installer enabled in the settings, the application silently switches the query engine to Pip when listing. The fast UV installer is used exclusively for the heavy lifting – installation and deletion.

4.2 Single Package Management (Interactive Panel)

The right panel is context-sensitive. It is activated only after clicking a specific row in the table (e.g., the requests package).

Intelligent "Update" Button

  • Something to update: If the versions differ, the button will be active with the text: "⬆️ Update to [version]".
  • Up to date: If you have the latest version, the button will be disabled and display: "✅ Version is up to date".

Uninstallation Safety Mechanism

The ❌ Uninstall button starts a destructive process. Therefore, the application will always stop you with a confirmation dialog (Yes/No) to prevent accidental deletion of an important library.

Live Birth Certificate Update

The system thinks of everything for you. With every successful installation, update, or uninstallation of a package (whether individually or in bulk), the application instantly runs the BirthCertificateGenerator process in the background. It automatically overwrites your venv_birth_certificate.json file. Thus, your environment's Birth Certificate always 100% reflects reality without you having to manually confirm anything.

4.3 Downgrade (Returning to an Older Version)

Sometimes the latest version of a package contains bugs or is incompatible with your code. In such cases, you need to perform a Downgrade.

In the right panel, under the "Install another version" section:

  1. Look at the "Installed version" column (e.g., 2.5.0).
  2. Type the lower version number you need into the text field (e.g., 2.4.0).
  3. Click the Install version button.

The application runs the command for installing the exact version in the background (e.g., pip install package==2.4.0). The installer automatically detects that you have a newer version, removes it first, and then installs the requested older version.

4.4 Bulk Update and Conflict Resolution

The ⬆️ Update all outdated button starts a batch process that tries to upgrade all red packages to the latest version at once. However, the behavior of this process differs significantly depending on which package manager you have set in the Application Settings.

Standard Manager (Classic Pip)

Classic pip tries to resolve dependencies during installation. However, if updating one package (e.g., lib_A to version 2.0) causes another package (lib_B) to require an older version (lib_A==1.0), a conflict known as "Dependency Hell" occurs.

What to do on Pip error: Manual Repair
Standard pip does not include a built-in command for quick and reliable environment re-checking. If the process fails or you suspect a conflict:
  1. Read the output in the Log window. Look for error messages like ResolutionImpossible or texts such as "ERROR: pip's dependency resolver does not currently take into account all the packages that are installed."
  2. Identify problematic packages. The log will often state which package requires which version.
  3. Perform a manual downgrade. In the right panel of the Pip Manager, find the package that needs to be reverted to an older version, enter the exact version number, and click "Install version".

Ultra-fast Manager (UV - Astral) ✨ Autopilot

If you use the modern uv manager in Settings, VenvHub Pro will launch an intelligent Autopilot for dependency checking and repair after every bulk update:

  1. Update: First, the latest versions of all outdated packages are downloaded and installed extremely quickly.
  2. Deep Check (uv pip check): Immediately after installation, the application verifies the integrity of the entire environment in the background.
  3. Auto-Fix: If any updated package has broken the dependencies of another (e.g., "The package X requires Y==1.5, but 2.0 is installed"), the application automatically reads this error from the log, forces the installation (downgrade) of the exact required version itself, and then performs the check again.
Result: By using uv, you get a maximally up-to-date yet 100% stable and error-free environment without having to manually decipher red logs and search for correct versions. The entire process runs on autopilot.

Is it possible to mix Pip and UV?

Yes, it is safe.

Both tools, pip and uv, work with the same standard package format and store them in the same location inside the virtual environment (in the Lib/site-packages folder). They are fully compatible with each other.

For example, you can:

  • Install a package using pip.
  • Subsequently run a bulk update using uv.
  • UV will recognize the packages installed by pip without issue, check them, and update or fix their dependencies if necessary.

Thanks to this compatibility, you can switch freely between package managers in the Settings depending on which one suits your situation better (e.g., UV for speed and repairs, Pip for maximum certainty and long-term stability).

4.5 Freeze and Logging

The 📜 Freeze button generates a requirements.txt file with the exact versions of all packages directly into the root of your project.

Logging and Safety Lock

All operations (Install, Update, Uninstall) run in their own thread (asynchronously), and their complete output is displayed in real-time in the bottom Log window. During any ongoing operation, the application deactivates (grays out) all buttons. This is an important safeguard against starting two installations at once, which could damage the virtual environment.
