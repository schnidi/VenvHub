12.1 Principle of Isolation and Portability

Standard VS Code stores all settings and extensions in two huge system folders in Windows (%APPDATA%\Code\User and %USERPROFILE%\.vscode\extensions). If you work on multiple projects with different requirements, or if you transfer work on a USB drive, this standard behavior is undesirable.

VenvHub Pro brings a "Portable" layer for VS Code:

  • Isolation: Each profile you create has its own folder. This means separate settings.json files, custom keybindings, and most importantly, physically separate lists of installed Extensions.
  • System Cleanliness: Your VS Code installation in Windows remains absolutely untouched. VenvHub Pro launches VS Code with special hidden parameters (--user-data-dir and --extensions-dir) that redirect it to your created folders.
  • Portability: If your profiles are on an external drive, your workspace (including all plugins) will be identical on every computer where the drive is connected.

12.2 Setting the Profile Root

For the feature to function, you must tell the application where to physically store all these profiles.

  1. Open Manager and go to the Settings tab.
  2. In the "VS Code Profile" section, click the Manage Accounts button.
  3. In the newly opened window, you will see the Profile Root Folder field. Click the folder icon 📁 and select an empty folder (e.g., vscode_users in your portable directory).
Intelligent Paths (UUID): Like other settings in VenvHub Pro, this path is protected by portable mode. If you change the drive letter (e.g., from E: to F:), the application automatically recalculates the path based on the unique drive ID, and the profiles remain functional.

12.3 Creating and Importing Profiles

In the right part of the "Manage VS Code Users" window, you can create new accounts.

Creating a New Profile

  • User ID: Technical name of the folder on disk (do not use spaces or diacritics, e.g., jozo_dev).
  • Display Name: Readable name that you will see in the list (e.g., "Jozef - Python Expert").

Importing from System

Creating an empty profile means starting VS Code from scratch (clean editor without theme or plugins). If you want your new profile to look exactly as it currently does in your Windows system, use the Import function. The application physically copies data from %APPDATA% and %USERPROFILE% directly into the profile folder.

  • Upon creation: Check "Copy from system upon creation," and the application will immediately start copying data after the folders are created.
  • Additionally: You can select an existing profile in the list at any time and click 📥 Import from system now (note that this will delete and replace the profile's previous settings with the system ones).
Intelligent Rollback in the background: Copying extensions can take up hundreds of megabytes and several minutes. VenvHub Pro performs this operation safely. If you press the "Cancel" button during the download, the application won't leave half-functional junk on the disk:

- If you cancel the import during the creation of a brand new profile, VenvHub will automatically delete the entire incomplete profile from the disk.
- If you cancel an additional import into an existing profile, VenvHub will only delete what it managed to copy and revert the profile to its original state.

12.4 Active Profile and "Triangular" Synchronization

VenvHub Pro introduces an advanced application state synchronization system. The goal is for the user to see the same data and the same "Master" state regardless of which window the change was made through.

1. Triple Synchronization Principle (Real-time Sync)

The application continuously connects three main nodes. If you change a VS Code profile in any of these places, the change is transferred to the other two within a fraction of a second (via signals and slots):

  • Quick Settings: ComboBox in the Mini-Bar slide-out panel.
  • Manager - Settings: Main ComboBox in the settings tab.
  • Manage Accounts (VSCode User Window): Profile list, where a star ★ [ACTIVE] appears and the text becomes bold for the active account.

2. Intelligent Venv Switching (Master Mode)

Every environment (Venv) change in Quick Settings (via Mini-Bar) behaves as a "Master" selection (Default):

  • Hard-Write to Config: The selected Venv is immediately written as primary for that project.
  • VS Code Sync: The system automatically opens .vscode/settings.json in the project and overwrites the interpreter path (python.defaultInterpreterPath).
  • Package Linker: If the selected Venv has local packages linked, the system immediately reports them to VS Code analysis (ExtraPaths).
  • Visual Response: If you have the Manager window open, you will see the star ★ in the table jump to the new row exactly when you click in the Mini-Bar.

3. VS Code Launch Button in Mini-Bar

The new VS Code icon on the main bar is linked to strict verification logic. What should the user expect?

  • Pre-start Check: Before launching, the system verifies if a project is selected, if the Venv physically exists on disk, and if its path is valid. If something is wrong, you receive a clear error message instead of a VS Code failure.
  • Automatic Profile Injection: If an active profile is set, the button automatically "bundles" it with the command. VS Code will open with your extensions and settings, even if it was previously open in the system under a completely different configuration.
  • Blocking in Multi-mode: The button is only available in Single mode. In Batch Launch (Multi) mode, it is automatically hidden to avoid confusion among multiple projects.
Important Rule for Portability: For portability to work correctly, never modify the Python path directly in VS Code. Always change it in the VenvHub Pro environment (in the Manager or Quick Settings). VenvHub checks and overwrites these paths according to the current drive letter every time VS Code starts. If you change it manually in the editor, the environment will likely break when transferred to another PC on a USB drive.
Always on top (Pin): The pin button 📌 on the Mini-Bar now controls both windows at once. If you pin the Mini-Bar, the Quick Settings panel is also automatically fixed on top of other windows.
