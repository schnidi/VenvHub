1. Installation and First Run

1.1 Installation Process

VenvHub Pro installation is designed to be as customizable as possible to fit your needs. After downloading and running the installer, the wizard will guide you through the following steps:

  1. Language Selection: Right at the start, you choose the installer language (this language may differ from the application language, which you can change later).
  2. Mode Selection: This is the most important step of the installation. The installer will ask if you want to perform a Classic Windows Installation, or if you want to extract the application as a Portable version. The differences are detailed below.
  3. Folder Selection and Completion: Choose the location and the installer will prepare all necessary files.

1.2 What is the difference between Classic and Portable versions?

VenvHub Pro includes intelligent environment detection. The way the application stores your settings, downloaded themes (skins), and other data depends exclusively on which type of installation you chose.

🏢 Classic Windows Installation

This mode is ideal for standard users who work on only one computer. The application is installed normally (for example, in C:\Program Files\VenvHubPro) and creates a shortcut in the Start menu.

  • Security: Since Windows protects the Program Files folder from overwriting, the application does not store your personal data there.
  • User Separation: If multiple users use one computer (having different Windows accounts), each will have their own settings and their own themes.

🧳 Portable Installation

This mode is intended for developers who want an "all-in-one" application. During this installation, a special file _is_portable.ini is created in the application folder, which serves as a flag for portable mode.

  • Independence: The application leaves almost no traces in the Windows system (no registry entries).
  • Mobility: You can copy the entire VenvHub Pro folder to a USB drive and use it outside the primary disk.

1.4 Project Transfer, Protection, and Repair Mechanisms

VenvHub Pro contains intelligent protection and repair mechanisms that prevent environment damage when transferring between different computers or when changing drive letters on USB.

Scenario A: Drive Letter Repair (Only for Embed venvs)

This mechanism is automatically activated at application startup if it runs in Portable mode (e.g., from a USB drive). Imagine you have USB connected as drive E: at home and as F: at work.

  1. The application detects at startup that it is running from disk F:.
  2. It loads paths to your portable (Embed) venvs, which still point to disk E:.
  3. Automatically and silently in the background, it overwrites all old drive letters with the new letter F:. It repairs not only its settings but also paths inside the activation scripts (e.g., activate.bat) of each portable environment.
System Venvs are protected: For system environments, the paths to Python (pointing to the system disk C:) are never overwritten or modified. They remain in their original state so that they are immediately unlocked upon returning to the home computer.

Scenario B: Hardware Lock and Blocking (System Venv)

This scenario occurs if you transfer a USB drive with system environments (which use the classic Python installed in your system on C:) to another computer (PC2), where this Python is not present or has a different version.

  1. When creating a system venv in Portable mode, the program quickly reads your computer's hardware fingerprint (formed from the PC name and the unique ComputerHardwareId identifier from the registries) and writes it to the portable_system_py.json file.
  2. If you connect the USB drive to another computer (PC2), VenvHub Pro immediately detects the hardware mismatch against the portable_system_py.json file.
  3. For security reasons (protection against C-library incompatibility and path corruption), the application completely blocks execution (Play), cloning, editing in VS Code, and access to the Pip manager for this venv.
Warning: A blocked system venv is not destroyed or modified. After reconnecting the USB drive to your original computer (PC1), the program verifies the hardware match and immediately unlocks the venv safely. On a foreign computer (PC2), you can only delete it or create a new clean portable (Embed) venv.

1.5 First Launch (Path Settings)

After successful installation (regardless of which mode you chose), launch the application. On the very first run, both the table and the project list will be empty. For the application to function at all, you must show it where you have your data on the disk.

Open the main Manager window (by clicking the 🖥️ icon or ⚙️ -> Settings) and set these two critical paths:

  1. Projects Root:
    Click the folder icon 📁 and select the main folder where you have all your Python projects stored (for example, D:\Python_Projects). The application will load all subfolders and treat them as separate projects.

  2. Venv Hub:
    This is the place where VenvHub Pro will physically create all virtual environments. We recommend creating a completely empty and separate folder specifically for this purpose (for example, C:\Venvs or D:\My_Venvs).
Only one instance: The application is designed to run only one copy at a time. If you try to run it a second time, the already running window will be brought to the foreground instead.
Pro-tip from practice: Never set "Venv Hub" directly into the folder with your projects! The application manages environments centrally (in one place) and links them to projects only logically. Mixing code and virtual environments would cause chaos.
