13. Complete integration with VS Code

VenvHub Pro does not just serve as a "launcher" for the editor. In the background, it contains extensive logic (in modules such as vs_code_json.py and system_sync.py) that actively configures Visual Studio Code. It ensures your editor is always ready to work with the correct dependencies, paths, and tools without you having to configure anything manually.

13.2 Safe merging in settings.json

VenvHub Pro never overwrites your entire configuration file. It uses an advanced non-destructive algorithm to manipulate .vscode/settings.json.

  • Parsing and preservation: The application reads the file first. It preserves all your existing personal settings (such as tab size, colors, or other plugin settings).
  • Applying the Interpreter: It clears outdated keys (e.g., python.venvPath) and sets the python.defaultInterpreterPath key exactly to the python.exe from your selected environment.
  • Fail-Safe mechanism: If your settings.json is corrupted (e.g., a missing bracket and cannot be loaded via json.loads), VenvHub won't crash. It silently creates a new, healthy structure so that IntelliSense doesn't fail.
  • Cleanup upon deletion: When you delete an environment from VenvHub that was set as "Default" for VS Code, the application automatically opens settings.json and removes the invalid path. If the file remains completely empty as a result, it deletes the entire file and the .vscode folder to avoid leaving clutter in your project.
Portability Rescue (USB Drives): If you move a project to another PC and the drive letter changes, VS Code would normally stop working. However, VenvHub Pro scans settings.json upon startup and fixes the old drive letter with the new one in a fraction of a second (via PortablePathLogic).

13.3 Feeding Pylance (Code Completion)

To ensure that imports don't show up in red (ModuleNotFoundError in VS Code) after adding your own local packages, VenvHub Pro ensures compatibility in two different ways, depending on the technology you use:

Chosen Technology How VenvHub links it to VS Code?
Module: Local Packages (Linker) When you link a package, VenvHub inserts its absolute path into the python.analysis.extraPaths array in settings.json. When unlinking, the application finds this path and safely removes it from the array without deleting your own (manually added) paths.
Module: Editable (pip -e) With this method, nothing is written to settings.json at all. Instead, Pip creates a .pth file inside the Venv. The Pylance analyzer in VS Code is intelligent enough to read this file natively. In this case, VenvHub "only" handles the automatic installation and drive letter repairs.

13.4 Meta-Sync (Injecting Tasks and Keybindings)

If you are creating advanced Python packages and tools, you sometimes need to add specific controls directly to the editor (e.g., running a script by pressing F5 or a terminal command in VS Code).

VenvHub Pro functions as a "Plugin Manager" for your editor. When you link a local package to a Venv using the Linker module, VenvHub examines its configuration in the background:

  1. It finds the local_meta.json file in your package.
  2. It looks for the special "vscode_integration" key.
  3. It takes all defined Tasks and writes them to .vscode/tasks.json in your project.
  4. It takes all Keybindings and writes them to .vscode/keybindings.json.
  5. Anti-duplication: The algorithm always verifies if the task already exists in the file before writing to avoid creating duplicates during multiple re-linking.
Power of Variables: Tasks injected this way are written universally in your local_meta.json. Thanks to the ${command:python.interpreterPath} variable, VS Code will always run the task using the Python Venv you assigned to it in VenvHub Pro!
