2.1 Description of Bar Elements (Mini-Bar)

The Mini-Bar is a floating bar designed to occupy minimum screen space while providing maximum control. Description of elements from left to right:

  • ● (Status Dot): Indicates the validity of the selected environment.
    • Green: The environment is OK and exists on the disk.
    • Red: The environment does not exist or its path is invalid.
  • INFO Text / Group List: Displays the name of the current project and Venv (in Single mode), or the group list (in Multi mode).

  • ▶ Play: Launches the assigned script or the entire group.
  • ■ Stop: Immediately terminates the running script or the entire group.
  • ⚙ (Gear): Opens/Closes the Quick Settings panel.
  • 🖥 (Monitor): Opens the Master Manager main window for advanced management.
  • 📌 (Pin): Toggles "Always on Top" mode. When active, the bar will overlay other windows (e.g., IDE).
  • M (Mode): Switch between Single mode (one project) and Multi mode (batch launching).
  • ✕ (Cross): Completely terminates the entire VenvHub Pro application.

2.2 Quick Settings (Slide-out panel)

Clicking the ⚙ icon slides out a panel that allows you to quickly change parameters without opening the Master Manager.

What can you set here?

  1. Project and Venv: Quick switching between projects and their environments.
  2. Entry Script: Displays which file will be launched. This path is configured in the Master Manager.
  3. Language and Appearance: Allows instant changes to the application's language and theme (skin).
  4. Quick Install (Pip/UV): Enter a package name (e.g., requests) and press Install. The package will be installed into the currently selected environment using your default package manager.

2.3 Temporary vs. Permanent Venv Change

This is a key feature of VenvHub Pro. The application distinguishes between what you set in Quick Settings and what you set in the Master Manager.

Temporary Change (Override) - Quick Settings

If you change the Venv in the Quick Settings panel, this change is temporary only (in RAM).

  • Used if you want to run a project in a different environment "just as a test".
  • This change will not be written to the configuration file.
  • After restarting the application, the project will revert to its default environment.

Permanent Change (Default) - Master Manager

If you want to permanently change the environment that should launch with the project, you must go to the Master Manager (🖥) window and check the "Default" box in the table.

Tip: In the Venv list within Quick Settings, the default (permanent) venv is marked with a star (★).

2.4 Launch Modes

In the "3. Actions" section, you can choose how the Play ▶ button behaves:

  • Open Terminal: Does not run the script; it only opens a command prompt (CMD) activated in that environment. Useful for manual command entry.
  • Run in Terminal (Default): Opens a new window, activates the venv, and automatically runs your script. You can see all outputs (prints) and errors.
  • Run in Background (Silent): The script runs without any window (invisibly). Suitable for final applications, GUI programs, or background processes.
Note for Silent Mode: Since you cannot see a window, you must terminate the script either via the Stop ■ button on the bar or via the Task Manager.

2.5 Multi-run Mode on Mini-Bar

When you click the M button, the Mini-Bar switches to batch launching mode. The interface changes:

  • Instead of the project text, a dropdown list appears with all your groups created in the Manager.
  • The ⚙ (Quick Settings) button is hidden because individual project settings have no significance in this mode.
  • The Play ▶ and Stop ■ buttons now control the entire selected group at once.
Advantage: In this mode, you don't need to open the main window. Simply select a group on the bar and, with one click, start or stop all servers and processes belonging to it.
