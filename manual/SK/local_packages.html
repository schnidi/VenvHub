10. Lokálne balíčky (Shared Libraries)

10.1 Princíp fungovania (Bez UAC)

Funkcia Lokálne balíčky rieši častý problém vývojárov: ako efektívne zdieľať vlastný kód (knižnice, moduly) medzi viacerými projektmi bez nutnosti ich neustáleho kopírovania alebo inštalovania.

VenvHub Pro na prepojenie nevyžaduje žiadne administrátorské práva (UAC) a nevytvára fyzické kópie ani zástupcov (Junctions) na disku. Namiesto toho využíva mimoriadne čistú, rýchlu a bezpečnú architektúru Python Import Hooks (sys.meta_path). Systém automaticky vyrieši:

  • Vytvorenie mapovacieho súboru venvhub.json v zložke site-packages cieľového Venvu.
  • Vloženie dynamického spúšťača (.pth a .py súbor), ktorý pri štarte Pythonu zachytí vaše importy a nasmeruje ich priamo k vašim kódom.
  • Automatickú tichú inštaláciu potrebných externých (PIP/UV) závislostí pre daný balíček.
  • Kompletné injektovanie absolútnych ciest a úloh do VS Code (IntelliSense).

10.2 Štruktúra balíčka (local_meta.json)

Aby VenvHub Pro vedel s vaším balíčkom pracovať, musí každý priečinok v archíve obsahovať konfiguračný súbor local_meta.json. Tento súbor definuje meno, verziu a to, čo balíček vyžaduje.

Príklad zápisu metadát:

{
    "name": "MojaDatabaza",
    "version": "1.2.0",
    "short_description": "Zabezpečuje pripojenie na MongoDB",
    "requires_local": ["MojLogger"],
    "requires_pip": ["colorama", "pymongo"],
    "vscode_integration": {
        "tasks": [
            {
                "label": "Zmazať Cache databázy",
                "type": "shell",
                "command": "python -c \"import db; db.clear()\""
            }
        ],
        "keybindings": [
            {
                "key": "ctrl+shift+d",
                "command": "workbench.action.tasks.runTask",
                "args": "Zmazať Cache databázy"
            }
        ]
    }
}
Kľúč Význam
short_description Krátky popis, ktorý sa zobrazí v tabuľke priamo v aplikácii VenvHub Pro.
requires_local Zoznam mien iných priečinkov v tom istom archíve, ktoré tento balíček potrebuje.
requires_pip Zoznam balíčkov z PyPI, ktoré sa majú automaticky doinštalovať.
vscode_integration Umožňuje definovať vlastné príkazy (Tasks) a klávesové skratky, ktoré sa automaticky pridajú do cieľového projektu.

10.3 Nastavenie koreňovej cesty

Pred prvým použitím musíte aplikácii povedať, kde sa nachádza váš archív lokálnych kódov:

  1. Otvorte Manager (🖥️) -> záložka Nastavenia.
  2. V sekcii Základné cesty nájdite položku Lokálne balíčky.
  3. Kliknite na ikonu priečinka 📁 a vyberte zložku, v ktorej máte svoje moduly (každý modul vo vlastnej podzložke).
Tip: Ak používate Portable verziu na USB kľúči, cesta sa automaticky prispôsobí aktuálnemu písmenu disku.

10.4 Prelinkovanie do Venvu

Samotné pridanie balíčka do konkrétneho prostredia prebieha nasledovne:

  1. V tabuľke prostredí kliknite pravým tlačidlom na vybraný Venv.
  2. Zvoľte možnosť 🔗 Lokálne balíčky.
  3. V otvorenom okne zaškrtnite balíčky, ktoré chcete v projekte používať.
  4. Kliknite na 💾 Uložiť zmeny (Prelinkovať).
Automatizácia závislostí: Ak zaškrtnete "Balíček A", ktorý vyžaduje "Balíček B", systém automaticky prečíta jeho `local_meta.json`, zapíše obe cesty do `venvhub.json` vo Venve a ticho doinštaluje všetky potrebné PIP balíčky.

10.5 Integrácia s VS Code

Jednou z hlavných výhod VenvHub Pro je koordinovaná synchronizácia s vývojovým prostredím VS Code. Keď urobíte zmenu v nastavení lokálnych balíčkov alebo prepnete aktívne virtuálne prostredie, systém vykoná nasledujúce operácie:

  • VenvHub Pro prečíta súbor venvhub.json v danom prostredí a extrahuje z neho absolútne cesty pripojených lokálnych knižníc.
  • Tieto cesty odošle do konfiguračného poľa python.analysis.extraPaths v súbore settings.json vášho VS Code projektu. Pylance a IntelliSense vďaka tomu okamžite rozpoznajú vaše importy a kód nebude vykazovať falošné chybové hlásenia.
  • Do projektu sa podľa konfigurácie v metadátach pripoja vlastné úlohy (tasks.json) a klávesové skratky (keybindings.json).
🛡️ Hybridný mechanizmus (Brutal Check) a súbor venvhub_tracker.json:

Aby VenvHub Pro pri odstraňovaní balíčkov omylom nezasiahol do úloh a skratiek, ktoré ste si vo VS Code vytvorili vy sami ručne, využíva kombinovaný overovací mechanizmus:

  1. Záznam o zápise (Účtenka): Pri každej integrácii úloh alebo skratiek si aplikácia zapíše presný zoznam vykonaných zmien do pomocného súboru .vscode/venvhub_tracker.json.
  2. Prioritné čítanie z disku: Pri odškrtnutí (deaktivácii) balíčka sa systém pokúsi prečítať zoznam úloh na vymazanie priamo z jeho local_meta.json na disku a korektne ich odstráni zo súborov VS Code.
  3. Záchranná brzda (Smazanie cez Prieskumník): Ak zistíte, že ste priečinok s balíčkom predtým zmazali ručne cez Windows Prieskumník (čiže jeho local_meta.json už neexistuje), VenvHub Pro automaticky použije dáta zo súboru venvhub_tracker.json, vyhľadá osirelé záznamy a bezpečne ich z konfigurácie VS Code vymaže.
  4. Zosúladenie pri prepínaní Venvov: Ak v rámci jedného projektu prepnete aktívne prostredie (napr. z Venvu A s lokálnymi balíčkami na Venv B bez balíčkov), synchronizátor na pozadí automaticky odstráni z editora úlohy patriace starému prostrediu a pripraví čisté rozhranie pre nové prostredie podľa jeho konfigurácie.
  5. Automatické upratovanie prázdnych konfigurácií: Ak po odinštalovaní balíčka alebo prepnutí Venvu nezostanú vo VS Code žiadne ďalšie aktívne úlohy či skratky, VenvHub Pro fyzicky odstráni súbory tasks.json, keybindings.json aj samotný venvhub_tracker.json, čím zabraňuje hromadeniu prázdneho balastu v priečinku projektu.

10.6 Pribalenie do PyInstallera (.EXE)

Vzhľadom na novú bez-UAC architektúru (Import Hooks) nie sú lokálne balíčky fyzicky prítomné v zložke prostredia. Klasický PyInstaller by ich pri statickej analýze kódu automaticky nenašiel.

VenvHub Auto-Injector (Plná automatizácia): Pri tvorbe finálneho .exe programu nemusíte v PyInstaller Builderi robiť ohľadom lokálnych balíčkov vôbec nič navyše!

VenvHub Pro Builder obsahuje inteligentný predprocesor. Pred spustením PyInstallera automaticky prečíta súbor venvhub.json vo vašom prostredí a sám na pozadí vygeneruje:
  • Parameter --paths, vďaka ktorému PyInstaller okamžite nájde všetok váš Python kód z lokálnych balíčkov (bez nutnosti vypisovať "Skryté importy").
  • Parameter --add-data pre každý jeden prepojený balíček. Aplikácia automaticky pribalí všetky vaše obrázky, .json konfigurácie a UI šablóny, ktoré sa nachádzajú vo vašich moduloch, priamo do výsledného .exe.

10.7 Samooprava ciest na USB kľúči (Portable režim)

Nová architektúra robí prenášanie aplikácie a projektov na USB kľúči extrémne spoľahlivým a používateľsky príjemným.

Blesková tichá samooprava: Vždy, keď aplikáciu spustíte na novom počítači:
  1. VenvHub Pro deteguje, že sa zmenilo písmeno disku (napr. z E:\ na F:\).
  2. Otvorí súbor venvhub.json v zložke vášho prostredia.
  3. Okamžite a úplne potichu v ňom prepíše staré písmená za nové.

Oproti staršej verzii (ktorá používala Windows Junctions) už na opravu nikdy nevyskočí UAC okno a proces prebehne za zlomok sekundy.

10.8 Sieťové disky (NAS a zdieľané zložky)

✅ Plná podpora pre umiestnenie kdekoľvek!

Obrovskou výhodou novej "Hook" architektúry (oproti zastaraným Pevným väzbám) je to, že váš "Koreň lokálnych balíčkov" môže byť umiestnený naozaj kdekoľvek.


Či už ide o váš lokálny disk, sieťový disk (NAS), alebo cloudovú zložku – všetko funguje bez problémov. Keďže VenvHub Pro zapisuje cesty iba ako bežný text do JSON súboru, operačnému systému (Windows/Linux) už nezáleží na tom, na akom type disku sa dáta fyzicky nachádzajú.
