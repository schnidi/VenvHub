12.1 Princíp izolácie a prenosnosti

Štandardný VS Code ukladá všetky nastavenia a rozšírenia do dvoch obrovských systémových priečinkov vo Windowse (%APPDATA%\Code\User a %USERPROFILE%\.vscode\extensions). Ak pracujete na viacerých projektoch s rôznymi požiadavkami, alebo ak prenášate prácu na USB kľúči, toto štandardné správanie je nežiaduce.

VenvHub Pro prináša "Portable" vrstvu pre VS Code:

  • Izolácia: Každý vami vytvorený profil má vlastnú zložku. Znamená to vlastné súbory settings.json, vlastné klávesové skratky a hlavne fyzicky vlastný zoznam nainštalovaných rozšírení (Extensions).
  • Čistota systému: Vaša inštalácia VS Code v systéme Windows zostáva absolútne nedotknutá. Aplikácia VenvHub Pro spúšťa VS Code so špeciálnymi skrytými parametrami (--user-data-dir a --extensions-dir), ktoré ho presmerujú do vami vytvorených zložiek.
  • Prenosnosť: Ak máte profily na externom disku, vaše pracovné prostredie (vrátane všetkých pluginov) bude identické na každom počítači, kde disk pripojíte.

12.2 Nastavenie koreňa profilov

Aby funkcia fungovala, musíte aplikácii povedať, kde má všetky tieto profily fyzicky ukladať.

  1. Otvorte Manager (Správca) a prejdite na záložku Nastavenia.
  2. V sekcii "VS Code Profil" kliknite na tlačidlo Správa kont.
  3. V novootvorenom okne uvidíte pole Koreňový priečinok profilov. Kliknite na ikonu 📁 a vyberte prázdny priečinok (napr. vscode_users vo vašom portable adresári).
Inteligentné cesty (UUID): Rovnako ako pri iných nastaveniach vo VenvHub Pro, aj táto cesta je chránená prenosným režimom. Ak zmeníte písmeno disku (napr. z E: na F:), aplikácia automaticky prepočíta cestu podľa unikátneho ID disku a profily zostanú funkčné.

12.3 Tvorba a Import profilov

V pravej časti okna "Správa VS Code Používateľov" môžete vytvárať nové účty.

Vytvorenie nového profilu

  • ID Používateľa: Technický názov zložky na disku (nepoužívajte medzery ani diakritiku, napr. jozo_dev).
  • Zobrazované meno: Čitateľný názov, ktorý uvidíte v zozname (napr. "Jozef - Python Expert").

Import zo systému

Vytvoriť prázdny profil znamená začať s VS Code úplne od nuly (čistý editor bez témy a pluginov). Ak však chcete mať v novom profile všetko presne tak, ako to už máte v aktuálnom Windowse, použite funkciu Import. Aplikácia fyzicky skopíruje dáta z %APPDATA% a %USERPROFILE% priamo do zložky profilu.

  • Pri vytvorení: Zaškrtnite "Pri vytvorení kopírovať zo systému" a aplikácia po vytvorení zložiek okamžite spustí kopírovanie dát.
  • Dodatočne: Kedykoľvek môžete vybrať existujúci profil v zozname a kliknúť na 📥 Importovať zo systému teraz (pozor, pôvodné nastavenia profilu sa tým vymažú a nahradia sa systémovými).
Inteligentné zrušenie (Rollback) na pozadí: Kopírovanie rozšírení môže zabrať stovky megabajtov a trvať niekoľko minút. VenvHub Pro túto operáciu vykonáva bezpečne. Ak počas sťahovania stlačíte tlačidlo "Zrušiť", aplikácia nenechá na disku polofunkčný odpad:

- Ak ste import zrušili pri vytváraní úplne nového profilu, VenvHub tento neúplný profil automaticky celý zmaže z disku.
- Ak ste zrušili dodatočný import do existujúceho profilu, VenvHub vymaže len to, čo stihol nakopírovať, a vráti profil do pôvodného stavu.

12.4 Aktívny profil a "Trojuholníková" synchronizácia

VenvHub Pro zavádza pokročilý systém synchronizácie stavu aplikácie. Cieľom je, aby užívateľ videl rovnaké dáta a rovnaký "Master" stav bez ohľadu na to, cez ktoré okno zmenu vykonal.

1. Princíp trojitej synchronizácie (Real-time Sync)

Aplikácia nepretržite prepája tri hlavné uzly. Ak zmeníte VS Code profil na ktoromkoľvek z týchto miest, zmena sa v zlomku sekundy (cez signály a sloty) prenesie do ostatných dvoch:

  • Rýchle nastavenia (Quick Settings): ComboBox v rolovacom paneli Mini-Baru.
  • Správca (Manager) - Nastavenia: Hlavný ComboBox v záložke nastavení.
  • Správa účtov (VSCode User Window): Zoznam profilov, kde sa pri aktívnom konte zobrazí hviezdička ★ [AKTÍVNY] a text stučnie.

2. Inteligentné prepínanie Venvu (Master Mode)

Každá zmena prostredia (Venv) v Rýchlych nastaveniach (cez Mini-Bar) sa správa ako voľba "Master" (Predvolený):

  • Hard-Zápis do Configu: Zvolený Venv sa okamžite zapíše ako hlavný pre daný projekt.
  • Synchronizácia VS Code: Systém automaticky otvorí .vscode/settings.json v projekte a prepíše cestu k interpretu (python.defaultInterpreterPath).
  • Package Linker: Ak má zvolený Venv pripojené lokálne balíčky, systém ich okamžite nahlási do analýzy VS Code (ExtraPaths).
  • Vizuálna odozva: Ak máte otvorené okno Správcu, uvidíte, ako hviezdička ★ v tabuľke preskočila na nový riadok presne v momente, keď ste klikli v MiniBare.

3. Spúšťacie tlačidlo VS Code v Mini-Bare

Nová ikona VS Code na hlavnom paneli je napojená na prísnu overovaciu logiku. Čo má užívateľ očakávať?

  • Pred-štartová kontrola: Systém pred spustením preverí, či je vybraný projekt, či Venv fyzicky existuje na disku a či je cesta k nemu validná. Ak niečo nesedí, namiesto zlyhania VS Code dostanete jasné chybové hlásenie.
  • Automatická injekcia profilu: Ak je nastavený aktívny profil, tlačidlo ho automaticky "pribalí" k príkazu. VS Code sa otvorí s vašimi rozšíreniami a nastaveniami, aj keby ste ho predtým mali otvorený v systéme úplne inak.
  • Blokovanie v Multi-režime: Tlačidlo je dostupné len v režime Single. V režime Hromadného spúšťania (Multi) sa automaticky skryje, aby nedošlo k zámene viacerých projektov.
Dôležité pravidlo pre Prenositeľnosť: Pre správne fungovanie prenosnosti nikdy neupravujte cestu k Pythonu priamo vo VS Code. Vždy ju zmeňte v prostredí VenvHub Pro (v Správcovi alebo Rýchlych nastaveniach). VenvHub pri každom štarte VS Code tieto cesty kontroluje a prepisuje podľa aktuálneho písmena disku. Ak to zmeníte manuálne priamo v editore, pri prenesení na USB kľúči na iné PC sa vám prostredie pravdepodobne rozbije.
Vždy na vrchu (Pin): Tlačidlo špendlíka 📌 na Mini-Bare teraz ovláda obe okná naraz. Ak pripnete Mini-Bar, automaticky sa zafixuje nad ostatnými oknami aj panel Rýchlych nastavení.
