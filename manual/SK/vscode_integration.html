13. Kompletná integrácia s VS Code

VenvHub Pro neslúži len ako "spúšťač" editora. Na pozadí obsahuje rozsiahlu logiku (v moduloch ako vs_code_json.py a system_sync.py), ktorá Visual Studio Code aktívne konfiguruje. Stará sa o to, aby bol váš editor vždy pripravený pracovať s tými správnymi závislosťami, cestami a nástrojmi, bez toho, aby ste museli čokoľvek konfigurovať ručne.

13.2 Bezpečné zlučovanie (Merge) v settings.json

VenvHub Pro nikdy neprepíše celý váš konfiguračný súbor. Na manipuláciu s .vscode/settings.json používa pokročilý ne-deštruktívny algoritmus.

  • Naparsovanie a zachovanie: Aplikácia si najprv súbor prečíta. Zachová všetky vaše doterajšie osobné nastavenia (ako veľkosť tabulátorov, farby, nastavenia iných pluginov).
  • Aplikovanie Interpretra: Premaže zastarané kľúče (napr. python.venvPath) a nastaví kľúč python.defaultInterpreterPath presne na python.exe z vami vybraného prostredia.
  • Fail-Safe mechanizmus: Ak je váš settings.json poškodený (chýba zátvorka a nedá sa načítať cez json.loads), VenvHub aplikáciu nezhodí. Potichu vytvorí novú, zdravú štruktúru, aby IntelliSense nezlyhalo.
  • Upratovanie pri mazaní: Keď z VenvHubu zmažete prostredie, ktoré bolo pre VS Code "Predvolené", aplikácia sama otvorí settings.json a túto neplatnú cestu z neho zmaže. Ak po nej zostane súbor úplne prázdny, zmaže celý súbor a zložku .vscode, aby vám v projekte nenechávala neporiadok.
Záchrana prenosnosti (USB Kľúče): Ak projekt prenesiete na iný PC a zmení sa písmeno disku, VS Code by za normálnych okolností prestal fungovať. VenvHub Pro však pri štarte preskenuje settings.json a v zlomku sekundy opraví staré písmeno disku za nové (cez PortablePathLogic).

13.3 Kŕmenie Pylance (Našepkávanie kódu)

Aby vám po pridaní vlastných lokálnych balíčkov nesvietili importy na červeno (ModuleNotFoundError vo VS Code), VenvHub Pro zabezpečuje kompatibilitu dvomi rôznymi spôsobmi, podľa toho, akú technológiu použijete:

Zvolená technológia Ako s ňou VenvHub prepája VS Code?
Modul: Lokálne balíčky (Linker) Keď pripojíte balíček, VenvHub vloží jeho absolútnu cestu do poľa python.analysis.extraPaths v settings.json. Pri odpojení balíčka aplikácia túto cestu nájde a z poľa ju bezpečne vymaže bez toho, aby vymazala vaše vlastné (ručne pridané) cesty.
Modul: Editovateľné (pip -e) Pri tejto metóde sa do settings.json nezapisuje vôbec nič. Namiesto toho Pip vo vnútri Venvu vytvorí .pth súbor. Analyzátor Pylance vo VS Code je natoľko inteligentný, že si tento súbor natívne prečíta sám. VenvHub sa v tomto prípade stará "len" o automatickú inštaláciu a opravu písmen diskov.

13.4 Meta-Sync (Injektovanie Tasks a Keybindings)

Ak tvoríte pokročilé Python balíčky a nástroje, niekedy k nim potrebujete pridať aj špecifické ovládanie priamo do editora (napr. spúšťanie scriptu stlačením klávesy F5 alebo príkaz v termináli VS Code).

VenvHub Pro funguje ako "Plugin Manager" pre váš editor. Keď pomocou modulu Linker pripojíte do Venvu lokálny balíček, VenvHub na pozadí preskúma jeho konfiguráciu:

  1. Nájde vo vašom balíčku súbor local_meta.json.
  2. Hľadá v ňom špeciálny kľúč "vscode_integration".
  3. Vezme všetky zadefinované Tasks (Úlohy) a zapíše ich do .vscode/tasks.json vo vašom projekte.
  4. Vezme všetky Keybindings (Skratky) a zapíše ich do .vscode/keybindings.json.
  5. Anti-duplikácia: Algoritmus pred zápisom vždy preverí, či sa daná úloha v súbore už nenachádza, aby nevytváral duplicity pri viacnásobnom prelinkovaní.
Sila premenných: Takto injektované úlohy sú vo vašom local_meta.json písané univerzálne. Vďaka premenej ${command:python.interpreterPath} VS Code vždy spustí danú úlohu pod tým Python Venvom, ktorý ste mu vo VenvHub Pro priradili!
